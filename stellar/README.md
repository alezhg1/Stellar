## 🎯 О ПРОЕКТЕ

Stellar — это мобильное PWA-приложение с AI-репетитором, которое не даёт готовые ответы, а учит мыслить: находит пробелы в знаниях, объясняет ошибки и строит персональный путь к результату ЕГЭ/ОГЭ.

### Ключевые особенности MVP:
- **Socratic Orchestrator** — AI никогда не даёт готовый ответ, использует scaffolding
- **Semantic Knowledge Graph** — интерактивная карта тем с подсветкой корневых пробелов
- **Predictive Progress** — ML-прогноз итогового балла на основе темпа и усвоения
- **Онбординг-диагностика** — 15 вопросов → карта пробелов → план на неделю

### Socratic Orchestrator: реализация через LangGraph

**Системный промпт (сокращённо):**
```
Ты — Socratic AI-репетитор. НИКОГДА не давай готовый ответ.
1. Определи тему и код ФИПИ
2. Задай наводящий вопрос
3. Проверь понимание
4. Если ошибка — найди корневую причину
5. Хвали за правильные рассуждения
```

**LangGraph nodes (псевдокод):**
```typescript
const workflow = new StateGraph({
  nodes: {
    analyze: analyzeQuestion,      // Извлечение темы, ФИПИ
    scaffold: generateHint,        // Генерация подсказки
    check: verifyUnderstanding,    // Проверка ответа ученика
    reflect: identifyRootCause,    // Поиск пробела в пререквизитах
  },
  edges: {
    analyze → scaffold → check → reflect → scaffold (цикл)
  }
});
```

**Полный пример промпта в `/app/api/chat/route.ts`:**

### Predictive Progress: модель прогноза

**Логика (упрощённая регрессия):**
```typescript
// currentScore = mastery * 80 + consistencyBonus
const avgMastery = topics.reduce((sum, t) => sum + t.accuracy, 0) / topics.length;
const studyDays = uniqueDaysFromSessions(sessions);
const consistencyBonus = Math.min(studyDays / 20, 1) * 10;
const currentScore = Math.round(avgMastery * 80 + consistencyBonus);

// predictedScore = current + weeklyImprovement * 4 недели
const weeklyImprovement = currentScore / Math.max(studyDays, 1) * 0.5;
const predictedScore = Math.min(currentScore + weeklyImprovement * 4, 100);

// daysToGoal = (target - current) / weeklyImprovement
const daysToGoal = (80 - currentScore) / weeklyImprovement;
```

**Где считается:** 
- Серверлес-функция `/api/progress/predict` (кеширование на 24 часа)
- Или клиентский расчёт (для MVP достаточно)

### Mastery Map: граф тем + pgvector

**Структура данных:**
```sql
CREATE TABLE topics (
  id UUID PRIMARY KEY,
  fipi_code TEXT UNIQUE,
  name TEXT,
  prerequisites UUID[],  -- Ссылки на родительские темы
  embedding vector(768)  -- Для семантического поиска
);

CREATE TABLE user_topic_mastery (
  user_id UUID,
  topic_id UUID,
  accuracy DECIMAL,  -- 0..1
  attempts INTEGER
);
```

**Алгоритм построения графа:**
1. Загрузить все темы с `prerequisites`
2. Для каждой темы вычислить `mastery` из `user_topic_mastery`
3. Построить DAG (Directed Acyclic Graph)
4. Визуализировать через SVG или React Flow
5. Красным подсветить темы с mastery < 40%

**Rollup-контекст для экономии токенов:**
- Вместо всей истории чата sending only:
  - Последние 3 сообщения
  - Summary предыдущего диалога (генерируется раз в 10 сообщений)
  - Текущий mastery по теме

### Анти-галлюцинации: привязка к ФИПИ

**Методы:**
1. **Few-shot промпты** с примерами правильных объяснений по кодификатору
2. **RAG (Retrieval-Augmented Generation):**
   - Перед генерацией ответа поиск в Supabase похожих задач по вектору
   - Добавление найденных решений в контекст промпта
3. **Post-processing:**
   - Regex-валидация кодов ФИПИ в ответе
   - Блокировка ответов с уверенностью < 70% (через logprobs)
4. **Human feedback:**
   - Кнопка «Ответ неверный» → сохранение в `chat_logs` с флагом
   - Еженедельный обзор модератором
