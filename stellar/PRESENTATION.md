## СЛАЙД 2: Проблема

**Заголовок:** Почему школьники не могут эффективно подготовиться к ЕГЭ?

**Текст:**
- **ЦА:** Школьники 8–11 классов (14–17 лет), готовящиеся к ОГЭ/ЕГЭ самостоятельно или с репетитором
- **Проблема:** Ученик не понимает, с чего начать, не получает объяснений ошибок, теряет мотивацию без видимого прогресса
- **Почему существующие решения не работают:**
  - YouTube/банки заданий — только тесты, нет обратной связи
  - Репетиторы — дорого (1500–3000₽/час), недоступно в регионах
  - Готовые приложения — «правильно/неправильно», без объяснений
- **Если проблема не решена:** Стресс, низкие баллы, потеря бюджетных мест, уверенность «я не способен»

**Визуал:** Инфографика «Путь ученика»: незнание → ошибки → стресс → низкий балл

**Критерии:** Проблемное заявление (10), Анализ актуальности (10)

## СЛАЙД 3: Актуальность и масштаб

**Заголовок:** Почему это важно сейчас?

**Текст:**
- **950 000+** школьников ежегодно сдают ЕГЭ в России (ФИПИ, 2024)
- **60%** выпускников занимаются с репетиторами, но доступ неравномерен:
  - Москва/СПб: 75% имеют репетитора
  - Регионы: 35–40%, особенно сельские школы (ВШЭ, 2023)
- **Средняя стоимость** подготовки по 2 предметам: 60 000–100 000₽/год
- **Почему сейчас:** AI-модели уровня GPT-4 бесплатны через OpenRouter, мобильные PWA заменяют нативные приложения
- **Риски старого подхода:** Потеря года, платное обучение, психологические травмы от «провала»

**Визуал:** Карта России с распределением доступа к репетиторам, график роста EdTech-рынка

**Критерии:** Анализ актуальности (10), Доп. баллы за исследования (5)

**Источники:**

## СЛАЙД 4: Решение — MVP

**Заголовок:** Stellar: AI-тьютор, который учит думать

**Текст:**
**Ключевые функции (MVP):**

1. **Socratic Orchestrator** — AI задаёт наводящие вопросы, разбивает задачу на шаги, никогда не даёт готовый ответ
2. **Semantic Knowledge Graph** — интерактивная карта тем: ошибка по тригонометрии подсвечивает корень (алгебра, геометрия)
3. **Predictive Progress** — прогноз балла ЕГЭ на основе темпа, точности, усвоения концепций
4. **Онбординг-диагностика** — 10 вопросов → карта пробелов → план на неделю

**Почему эти функции:** Максимум ценности при минимуме кода, реализуемо за 1–2 месяца командой 2–3 человека

**Визуал:** Схема 4 блоков с иконками, стрелки к «Результат: +15 баллов»

**Критерии:** Качество решения (10)


## СЛАЙД 6: Технологический стек

**Заголовок:** Почему PWA + Next.js + Supabase?

**Текст:**
- **Платформа:** Веб-PWA (адаптивный, mobile-first)
  - 100% покрытие устройств (iOS, Android, Desktop)
  - Нулевая установка (добавляется на домашний экран)
  - Быстрый деплой на Vercel (секунды)
  - Работает в РФ без блокировок
- **Фронтенд:** React + TypeScript (Next.js App Router)
- **Бэкенд:** Vercel Serverless API Routes (Node.js)
- **БД:** Supabase (PostgreSQL + Auth + pgvector)
  - EU-регион для стабильности
  - Векторный поиск для семантики
- **AI:** OpenRouter API (бесплатные модели: Mistral-7B, Llama-3)
- **Оркестрация:** LangGraph/LangChain для scaffolding

**Визуал:** Схема архитектуры: Frontend → API → LangGraph → OpenRouter → Supabase

**Критерии:** Качество решения (10)

## СЛАЙД 7: Ключевые экраны

**Заголовок:** 3 главных экрана MVP

**1. Онбординг (Диагностика)**

- Взаимодействие: Клик по теме → фильтрация чата
- Что дальше: Диалог с AI → решение задач

**3. Чат с AI (Socratic Tutor)**
- Этап: Обучение
- Элементы: История сообщений, поле ввода, подсказки
- Взаимодействие: Ввод вопроса → streaming-ответ AI
- Что дальше: Обновление карты знаний → рост прогноза

**Визуал:** Мокапы 3 экранов в стиле Figma (светлая тема, минимализм)

## СЛАЙД 9: Риски и митигация

**Заголовок:** Главный технологический риск

**Текст:**
**Риск:** AI-галлюцинации (модель выдумывает формулы/ответы)

**Последствия:** Ученик запоминает неверную информацию, теряет доверие

**План митигации:**
1. **RAG (Retrieval-Augmented Generation):** База знаний ФИПИ в pgvector, AI отвечает только по контексту
2. **Системный промпт:** Жёсткие правила («Не давай ответ», «Сверяйся с кодификатором»)
3. **Валидация ответов:** Второй AI-агент проверяет ответ перед отправкой
4. **User feedback:** Кнопка «Ответ неверный» → логирование → дообучение
5. **Fallback:** Если неуверенность >30% → «Я не уверен, давай сверимся с учебником»

**Визуал:** Схема RAG: Запрос → Поиск в БД → Контекст → AI → Ответ

**Критерии:** Потенциал решения (доп. 5)
